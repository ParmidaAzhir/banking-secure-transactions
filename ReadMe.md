## Running the Project

The project demonstrates six scenarios covering the core security properties of **integrity, confidentiality, authenticity, and trust**.

---

## 1) Valid Secure Transaction

This scenario demonstrates the complete secure workflow:

* Create a JWT from the transaction data.
* Sign the JWT using the sender's private key.
* Encrypt the JWT for the receiver.
* Decrypt the packet and verify both the sender's certificate and the JWT signature.

**Expected Result:** `VALID`

```bash
python src/jwt_issue.py data/transaction_data.json keys/sender_key.pem data/token.jwt

python src/encrypt_packet.py data/token.jwt keys/receiver_cert.pem data/packet.json

python src/decrypt_and_verify.py data/packet.json keys/receiver_key.pem keys/sender_cert.pem keys/ca_cert.pem
```

---

## 2) Tampered Transaction Data

Modify the `amount` field in `transaction_data.json` **after** the data has been signed.

This demonstrates that even a small modification invalidates the digital signature.

**Security Property:** Integrity

**Expected Result:** `INVALID`

```bash
python src/verify.py data/transaction_data.json data/signature.bin keys/sender_cert.pem keys/ca_cert.pem
```

Restore the original transaction data before continuing.

---

## 3) Wrong Receiver Key

Attempt to decrypt the encrypted packet using an unauthorized receiver's private key.

The packet cannot be decrypted because it was encrypted using the legitimate receiver's public key.

**Security Property:** Confidentiality

**Expected Result:** Decryption fails.

```bash
python src/decrypt_and_verify.py data/packet.json keys/fake_receiver_key.pem keys/sender_cert.pem keys/ca_cert.pem
```

---

## 4) Expired JWT

Temporarily set a very short expiration time in `jwt_issue.py`, generate a new JWT, and verify it after it expires.

The token verification fails because the expiration time has passed.

**Security Property:** Authentication / Token Validity

**Expected Result:** JWT expired.

```bash
python src/jwt_issue.py data/transaction_data.json keys/sender_key.pem data/token.jwt

python src/jwt_verify.py data/token.jwt keys/sender_cert.pem keys/ca_cert.pem
```

Restore the original expiration time after testing.

---

## 5) Fake Sender

Sign the transaction using a fake sender's private key while attempting to verify it with the legitimate sender's certificate.

Verification fails because the signature was not generated by the trusted sender.

**Security Property:** Authenticity

**Expected Result:** `INVALID`

```bash
# Sign using the fake sender's private key
python src/sign.py data/transaction_data.json keys/fake_sender_key.pem data/signature.bin

# Verify using the legitimate sender's certificate
python src/verify.py data/transaction_data.json data/signature.bin keys/sender_cert.pem keys/ca_cert.pem
```

Restore the original signature:

```bash
python src/sign.py data/transaction_data.json keys/sender_key.pem data/signature.bin
```

---

## 6) Fake Certificate Authority (CA)

Verify a JWT that was signed using a sender certificate issued by a fake Certificate Authority.

Verification fails because the sender's certificate cannot be validated against the trusted CA certificate.

**Security Property:** Trust / Certificate Validation

**Expected Result:** Certificate verification fails.

```bash
python src/jwt_verify.py data/fake_token.jwt keys/fake_sender_cert.pem keys/ca_cert.pem
```
